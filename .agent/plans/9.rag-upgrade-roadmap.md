# 9. RAG Upgrade Roadmap — Multimodal + Smarter Retrieval

🔴 **Complex** — multi-phase program. Break each phase into its own sub-plan before executing. Phase 0 is a hard gate (verify live DB + close security holes) and must precede everything.

> **Provenance.** Produced by a 28-agent research+audit workflow (5 code auditors, 10 web-research topics each adversarially fact-checked, then synthesize→critique→revise). Security/code claims below marked **[verified]** were re-read directly against source by the main agent. DB-schema claims (HNSW index, RPC bodies) are **[assumed — confirm in Phase 0]** because the live `chunks`/RPC DDL was applied via the Supabase dashboard and is NOT in `backend/migrations/` (only `001`, `007`, `008` are checked in).

## Verified findings (re-read against source)
- **[verified] JWT signature verification disabled** — `dependencies.py:31` `options={"verify_signature": False}`. Backend runs the service-role client (RLS bypassed), so a forged `sub` fully impersonates any user. **Release-blocker.**
- **[verified] SQL tool not scoped** — `sql_service.py:88-93` `_inject_user_filter` returns SQL unchanged whenever `user_id in sql` (almost always true). Isolation rests entirely on the LLM choosing to write the `WHERE user_id=...` clause; no enforced backstop. RPC `execute_readonly_query` is `SECURITY DEFINER`. **Cross-tenant read risk.**
- **[verified] Decryption fails open** — `settings.py:40` catches `(InvalidToken, Exception)` and returns ciphertext as-is; `get_fernet()` returns `None` with no key → silent plaintext storage.
- **[verified] Unconditional `dimensions=`** — `embedding_service.py:54` sends `dimensions=` on every embed call; breaks any non-OpenAI embedder (the provider-swap the multimodal plan depends on).
- **[assumed — confirm] Probable missing vector index** — every search may be a sequential scan; confirm via `SELECT indexdef FROM pg_indexes WHERE tablename='chunks'`.

## 1. Executive Summary
- Well-built single-vector **text** RAG (HyDE → RRF hybrid → LLM rerank) that is 100% text-blind: no image upload, PDFs parsed text-only via PyPDF2 (charts/figures/scans dropped), no vision call anywhere, chat carries no images in or out. The default model is vision-capable and simply never shown a pixel.
- **Highest-leverage move: "caption-then-embed" multimodal + vision-in/image-out chat.** Render PDF pages / accept image uploads → describe with the vision model → embed the description through the *existing* text pipeline → store the source image in Storage. Reuses pgvector, RLS, hybrid search, reranker unchanged; **no new embedding provider**; respects every hard rule. Preferred over native multimodal embeddings or ColPali.
- **Two chat-layer bugs block multimodality (medium effort, not low):** (a) history rebuilt text-only (`select("role, content")`, `chat.py`) drops persisted `tool_calls`/`role:tool`/images on reload — defeats the stateless-completions rule; (b) `messages.content` is a hard `str`, so the OpenAI vision content-parts shape can never be built or replayed. Touches SSE, persistence, replay, streaming tool-call reassembly + a data migration of existing rows.
- **Backend runs on the service-role client → RLS is inert on every table path.** Isolation rests on app-level `.eq('user_id', …)` filters + an unverified JWT. "RLS on every table" is defense-in-depth that is currently bypassed, not live protection. Release-blockers: disabled JWT verification; unscoped `execute_readonly_query`.
- **Retrieval quick wins (confirm against Phase-0 DDL):** likely-missing vector index (seq scan per search); HyDE + LLM-reranker = 3 sequential LLM round-trips per search; reranker truncates passages to 500 chars (chunks are 1000); `tool_executor` mislabels an RRF score as `similarity` with a latent `KeyError`. **Confirmed:** unconditional `dimensions=`.
- **"Smarter" is mostly recombining parts you already have.** For a single user, latency is the constraint → an adaptive router is the gate that keeps heavy paths rare and mutually exclusive. Promote parent-document retrieval + chunk-aware contextual embeddings to Phase 1. Isolate CRAG/decomposition behind the `iterative` route. **Self-RAG reflection and RAPTOR are dropped** (most latency, least single-user value).
- **Stay single-vector and on managed Supabase.** ColPali/late-interaction and dedicated vector engines (Qdrant/VectorChord) are avoid/situational (break pgvector and/or managed Supabase). **Cohere Embed v4** (1536-dim) is the sanctioned Phase-7 upgrade only if caption recall plateaus — needs a **bespoke client**, not a `base_url` swap.
- **Eval is retrieval-only (recall@k, MRR).** Add reference-free generation judges (faithfulness, groundedness) + citation verification, reimplemented raw with Pydantic (not RAGAS, which pulls LangChain).

## 2. Current-State Assessment

### Strengths (keep)
- True RRF hybrid search (`match_chunks_hybrid`, FULL OUTER JOIN, k=60, over-fetch ×3). *(body assumed; confirm Phase 0)*
- Graceful degradation: HyDE/hybrid/rerank each in independent try/except.
- Contextual embeddings store the *raw* chunk (cited text + FTS unpolluted).
- Strict structured outputs everywhere (json_schema strict + Pydantic).
- Correct streaming tool-call reassembly (`tool_calls_buffer` keyed by `tc.index`; loop bounded; post-loop fallback).
- Storage RLS is real and load-bearing (private bucket, per-user folder RLS, signed-URL mint, sha256 dedup, Fernet keys masked to last-4).

### Highest-severity issues
**Release-blocking security** — disabled JWT verification; unscoped `execute_readonly_query` + no-op `_inject_user_filter`; `decrypt_value` echoes ciphertext on failure / silent plaintext; systemic: audit every service-role `.table()`/`.rpc()` for an explicit `user_id` filter (RLS is not the backstop).
**Confirmed code bug** — unconditional `dimensions=` (`embedding_service.py:54`).
**Multimodal blockers** — no image path on ingest; probable missing/invalid vector index *(confirm)*; chat string-only end to end (`tool_executor` flattens results to `[Source: filename]`, discarding `storage_path`/`chunk_id`/`document_id`).
**Cost/latency** — 3 sequential LLM calls per search; HyDE always on; `full_response` never reset between tool rounds (duplication); `r['similarity']` subscript + mislabeled RRF score.
**Process drift** — chunks/documents/RPC DDL not in `migrations/`.

## 3. THE MULTIMODAL UPGRADE
**Headline: caption-then-embed (VLM-as-parser) primary; vision-in / image-out in chat; single text-embedding space; signed-URL serving.** Cohere Embed v4 = Phase-7 (bespoke client). ColPali/late-interaction = out of scope.

| Approach | Verdict | Reasoning |
|---|---|---|
| **VLM caption-then-embed (DEFAULT)** | recommended | Reuses entire pipeline (caption → contextual-embed → text-embedding-3-small@1536 → hybrid → rerank). Zero new provider, no vector migration, default model already vision-capable. |
| **Native multimodal embeddings (Cohere Embed v4)** — Phase-7 fallback | situational | True image-to-image; 1536-dim fits the column with no migration. But NOT OpenAI-`/v1/embeddings`-shaped — needs a **bespoke client**, and adopting it means re-embedding the whole corpus. Adopt only if caption recall plateaus. |
| **ColPali / ColQwen2 late-interaction** | avoid | ~1024 patch vectors/page + MaxSim; pgvector has no MaxSim (needs Qdrant/Milvus/Vespa/VectorChord — not on managed Supabase); self-hosted GPU. Breaks the stack. |

Known limitation of the default: retrieval matches on the text description, not pixels — fine visual similarity is lost at the **search** step but regained at **generation** by attaching the real image to the vision model. Phase 7 closes the search-step gap if it becomes the bottleneck.

### (a) Ingestion
**License correction:** do NOT use PyMuPDF/`fitz` (AGPL-3.0). Use **`pypdfium2`** (BSD/Apache) or `pdf2image`+Poppler. (Docling optional Phase-2 for richer tables.)
1. Make extraction async-safe (`await asyncio.to_thread(...)` for `storage.download`, `extract_text`).
2. Add `image/*` branch to `extract_text` → `describe_image()` sends image as `image_url` base64 data-URL → Pydantic `VisionDescription{description, ocr_text, objects, chart_data}`. Chunkable text = `description + "\n" + ocr_text`.
3. PDF: per-page text + **gated, budgeted** page-image captioning. Change `_extract_pdf` to yield per-page `(text, {page_number})` segments. Caption candidate = empty/low text OR embedded raster. Store page PNG at `{user_id}/{document_id}/p{n}.png`; emit description chunk with `page_number`+`image_path`.
   - **Cost budget (mandatory):** cap captioned pages/doc (e.g. first 20 + sample); two-pass detail (`low` triage → `high` only for charts/tables/dense text); `asyncio.Semaphore(4)`; push estimated cost via Realtime so the user can abort; documented worst case ≈ \$0.15–0.30 per 100-page scanned PDF (confirm vs Phase-0 pricing).
4. Chunk per-segment, carry provenance (`content_type` `'text'|'image'`, `image_path`, `page_number`); never chunk across pages; row-wise chunking for csv/xlsx/html tables (repeat header).
5. Allow image uploads (`image/png,jpeg,webp,gif`) in `documents.py` allowlists + size caps.
6. Fix `dimensions=` (conditional) and stamp actual returned model+dim into chunk metadata.

### (b) Storage + schema
Dump live DDL first (Phase 0). Then one migration **written against the dumped bodies**:
```sql
ALTER TABLE chunks ADD COLUMN content_type text NOT NULL DEFAULT 'text';  -- 'text' | 'image'
ALTER TABLE chunks ADD COLUMN image_path  text;
ALTER TABLE chunks ADD COLUMN page_number int;
ALTER TABLE documents ADD COLUMN has_images boolean NOT NULL DEFAULT false;
-- CONFIRM FIRST: if no index / unconstrained vector, standardize 1536 + HNSW:
ALTER TABLE chunks ALTER COLUMN embedding TYPE vector(1536);
CREATE INDEX idx_chunks_embedding ON chunks USING hnsw (embedding vector_cosine_ops);
-- add `AND d.status='completed'` to BOTH arms of match_chunks_hybrid (after confirming arm names)
```
Storage: reuse existing private bucket + per-user folder RLS; image chunks reuse `{user_id}/…` paths. No multi-vector needed for the default. **Signed-URL serving:** `GET /chunks/{chunk_id}/image-url` with explicit `user_id` ownership check (service role bypasses RLS) → `create_signed_url(image_path, 300)`. Mint short-lived; never persist URLs (persist `image_path`/`chunk_id`, resolve fresh).

### (c) Retrieval
Image chunks are text rows → already flow through HyDE → hybrid → rerank. Real work: **stop discarding identifiers.** Return structured dicts carrying `content_type/image_path/page_number/document_id/chunk_index`; fix `r.get('similarity', r.get('rrf_score', 0.0))`; relabel field `relevance`/`rerank_score`; add a post-rerank relevance floor (<3/10 dropped) so off-topic queries return "No relevant documents found."

### (d) Answer generation
Vision-in: widen `content` to `str | list[dict]` (jsonb); build user turns as content parts; persist image refs (not base64); `detail:'low'` for thumbnails/routing, `'high'` for charts. Feed retrieved image chunks to generation as `image_url` parts. Image-out: add `event: sources`/`event: image` SSE frames; `onSources` in `api.ts`; citations row in `ChatView.tsx`; persist citations as structured content; **sanitize images** (`SafeImg` allowlisting the Supabase Storage domain + `rehype-sanitize`) to stop LLM-emitted tracking-pixel beacons.

### (e) Model/API picks
Decouple roles, pin ids in Phase 0 (verify live OpenAI/OpenRouter list — do NOT inherit `gpt-4o` long-term):
- **Generation+vision model** — must be vision-capable; used for `describe_image()` + chat.
- **Router/judge model** — cheap, may be text-only; used for router, LLM-judge eval, triage.
Both config in `global_settings`. Vision via `image_url` works identically through OpenRouter to GPT/Gemini/Claude — `base_url` stays a real swap point **for chat/vision** (not embeddings). Vision cost ≈ 85 base + 170/512px tile; ~765 tok per 1024² high-detail.

## 5. Quick-Win Fixes
1. **History replay (PREREQ, medium):** `select("role, content, tool_calls, tool_call_id")`, rebuild OpenAI-shaped messages (assistant `tool_calls` + matching `role:'tool'` ordered so each tool msg follows its call). Mis-ordering → hard 400s.
2. **`content: str | list[dict]` as jsonb (PREREQ, medium):** + data migration of existing rows + a test asserting a reloaded tool-call thread passes the provider without a 400.
3. Reset `full_response` per round (`chat.py`).
4. `tool_executor` score fix (kill `KeyError`, relabel `relevance`).
5. Gate HyDE for short/keyword/ID/filtered queries.
6. Reranker passage budget (stop truncating 1000-char chunks to 500).
7. `datetime.utcnow()` → `datetime.now(timezone.utc)`.
8. Async-safe ingestion (`asyncio.to_thread`); cap/warn above N chunks.
9. Wire or delete `expand_query`; add `AND d.status='completed'` to hybrid RPC (after Phase 0); pin `search_path` on SECURITY DEFINER functions.
10. Frontend: per-file upload results; structured 409 (not English substring); robust SSE frame parsing + inactivity timeout; reconcile temp/server ids.
11. **Conditional `dimensions=` (PREREQ for any non-OpenAI embedder):** send only for OpenAI `text-embedding-3-*`; stamp actual returned vector length.

## 6. Recommended Sequencing
**Phase 0 — Verify reality + secure (gate).** Dump live DB (`pg_indexes`, `\d+ chunks`, full bodies of `match_chunks_hybrid`/`match_chunks`/`execute_readonly_query`) → commit to `migrations/`; pin model ids; fix JWT verification, scope the SQL tool, fix `decrypt_value`, audit every service-role call for a `user_id` filter; ship conditional-`dimensions`. *Validation:* forged JWT → 401; SQL tool returns only caller's rows (`test@` vs `test2@`); startup fails without encryption key; `supabase db push` applies dumped DDL clean; non-OpenAI embedder works.
**Phase 1 — Retrieval correctness + speed + best-evidence quality.** HNSW (if absent) + 1536; dedicated reranker + fallback; gate HyDE; reranker full-passage + relevance floor; score fix; status filter; **parent-document + chunk-aware contextual embeddings.** *Validation:* `EXPLAIN` shows index scan; recall@5/10/MRR improves vs baseline (A/B); latency drops; off-topic → "No relevant documents found."
**Phase 2 — Chat-layer prerequisites (medium).** `content` jsonb + data migration; full history replay; reset `full_response`; `datetime`; SSE `sources`; `SafeImg` + `rehype-sanitize`. *Validation:* reload tool-call thread → passes provider (no 400) + no redundant re-search; no duplicated text; external-domain image renders as a link.
**Phase 3 — Multimodal ingestion.** `pypdfium2` rasterization + per-page segments; image uploads + `describe_image`; per-doc cost budget; new columns + `has_images`; signed-URL endpoint; async-safe; table row-wise chunking. *Validation:* chart-only PDF page → image chunk w/ caption + resolvable `image_path`; PNG searchable; 100-page scan respects cap + surfaces estimate, no 100 sequential high-detail calls; `test2@` can't mint URL for `test@`'s image.
**Phase 4 — Multimodal chat.** Image attach/paste → `image_url` parts; attach retrieved images to generation; `image`/`sources` SSE; persist+replay citations; thumbnails. *Validation:* paste screenshot → model references it; figure-answer query returns cited page image inline + survives reload; `detail:'low'` for thumbnails.
**Phase 5 — Smarter retrieval & eval (router-gated).** Adaptive router; iterative-only CRAG/decomposition/citation+groundedness; reference-free judges off hot path; free Moderation; nDCG@10. *Validation:* router accuracy on held-out set + chit-chat skips retrieval; simple route ≤2 LLM calls + p95 met; CRAG returns "not found" out-of-corpus; faithfulness logged with judge-vs-human agreement.
**Phase 6 — Memory & personalization (optional).** `user_memories` + `user_memory_blocks` (RLS + explicit filter); background extractor; profile injection; personal/work scoping. *Validation:* durable fact recalled in a new thread; personal-scoped memory doesn't surface in work; cross-user isolation holds.
**Phase 7 — Cohere Embed v4 (conditional).** Only if caption recall plateaus: additive `embedding_mm vector(1536)` + `match_chunks_mm` + RRF fusion + bespoke client. *Validation:* image-to-image retrieval beats caption-only on a held-out visual set; text retrieval unchanged.
